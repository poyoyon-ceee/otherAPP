Gitコマンドガイド

GitHubでのリポジトリ作成・クローン作業を効率化するためのコマンド集です。

新しいリポジトリを作成してプッシュする

PC1で新しいプロジェクトを始める際の手順です。ローカルに新しいGitリポジトリを作成し、GitHubにアップロードします。

kaunetAPP

git init
git remote add origin https://github.com/poyoyon-ceee/kaunetAPP.git
echo "# kaunetAPP" > README.md
git add .
git commit -m "初回コミット"
git branch -M main
git push -u origin main

GitとGitHubの関係性

Gitは、あなたのPC上で動作するバージョン管理ツールです。一方、GitHubは、インターネット上のリポジトリホスティングサービスです。この2つは役割が異なります。

なぜGitHubに先にリポジトリを作成するのか？

ローカルでgit initを実行すると、あなたのPC内に隠しフォルダ（.git）が作成され、ファイルの変更履歴が記録され始めます。しかし、この時点ではまだGitHubとは一切関係がありません。

git remote add originコマンドは、**「ローカルの.gitフォルダを、GitHubのこのURLと紐づけますよ」**と設定するために使います。したがって、紐づけ先のGitHubリポジトリが事前に存在している必要があります。

たとえるなら、GitHubに**「ファイルの保管場所」**を事前に用意しておき、ローカルからその場所にファイルを送り届ける、というイメージです。

⚠ 強制的にリモートを反映するコマンド（注意）

このコマンドは、**あなたのローカルでのすべての変更を破棄し、GitHubの最新の状態に強制的に合わせる**ためのものです。ローカルの未コミットの変更や、まだプッシュしていないコミットはすべて消去されます。

**緊急時や、ローカルの状態が壊れてしまった場合にのみ使用してください。**

git fetch origin
git reset --hard origin/main
git clean -fd
  • git fetch origin
    GitHubから最新の情報を取得しますが、ローカルには反映しません。
  • git reset --hard origin/main
    ローカルのブランチを、GitHubの最新コミットの状態に強制的に戻します。
  • git clean -fd
    ローカルの作業ディレクトリにある、Gitの管理下ではないファイルやフォルダをすべて削除します。
